# CSCI 3230U - AI Use Policy (Group Project)

AI coding tools are part of professional web development, and you should learn to use
them well. This policy lets you do that while keeping the early learning honest and
keeping no one at a disadvantage.

## Summary

1. **Learning with AI is always fine.** Use an AI assistant to explain a concept,
   interpret an error message, or talk through a design whenever you like - no
   disclosure needed. This is studying, not submitting.
2. **AI that produces submitted work must be documented.** If AI generated or edited any
   code you submit (in whole or in part), you must record it in `AI-USAGE.md`,
   including the exact prompt you used (see the template).
3. **Milestones 1–2 (weeks 3, 6) are AI-free.** Everything you submit and present up
   to and including the week-6 progress milestone must be written by your team
   without AI generating code. Milestone 3-4 (week 10, final exam week) may use
   AI.
4. **Free models only.** Use AI tools that do not require a paid plan - free tiers
   are fine (e.g. the free tiers of common assistants, or an education/free tier). No
   one should need to spend money to do this course. (We can't verify which model you
   used, but the rule stands and undocumented AI code is an integrity violation.)

## How to use AI well (the encouraged workflow)

- **Do it yourself first.** Write the feature by hand so you actually learn it
- **Then delegate the tedium.** Once a task is repetitive or boilerplate, that's the
  time to ask AI - and you'll be able to judge whether its answer is right
- **Always review and verify.** You own every line you submit. If you can't explain it,
  don't submit it - you may be asked to explain any part of your project

## What counts as "AI-generated"

| Documented in `AI-USAGE.md` | Not regulated |
|---|---|
| AI wrote a function/component you submit | AI explained how `useEffect` works |
| AI refactored or fixed your submitted code | AI helped you read an error message |
| AI generated CSS/markup you submit | AI suggested which library to research |
| Autocomplete that produced non-trivial blocks | Single-token editor completions |

## Documentation requirement

Keep an **`AI-USAGE.md`** at the root of your project repo. Add one entry every time
AI contributes to submitted code, before you commit that code. Each entry records
the date, the group member, the files touched, the verbatim prompt, and what you
kept or changed. Use `project/AI-USAGE-template.md` as the starting point. Add a short
marker comment where AI-assisted code lives, e.g. `// AI-assisted - see AI-USAGE.md #7`.

## Integrity

Submitting AI-generated code on an AI-free milestone, or AI-generated code without
documenting it on the final milestone, is academic misconduct and will be handled
under the university's academic-integrity policy. Documenting honestly is always the
safe choice - the documentation is a normal, professional habit.
